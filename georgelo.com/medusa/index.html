Medusa

Medusa is a Java tool that will show interactive classes, methods, and variables of a Java project. After the user chooses a large Java project (as a zip or a folder), Medusa would traverse into it, parse every single file inside, and make a interactive playground.

This group project (with 5 other teammates) was a class project for our software engineering class. We had scrum and sprint meetings to ensure that this program will be done in a semester.

The idea is that it should allow software engineers to quickly identify and visualize the relationships of various classes, methods, and variables. Especially in the field of software engineering, code base is always huge. So we thought this program would be extremely helpful to programmers who want to quickly understand a large project.

My responsibility was to setup diagram models, develop custom views, allow the user to save/load our project document file, and fix bugs. To elaborate more on our project document file, it is a JSON-based file that has all the classes, methods, and variables as well as where they should be placed (as nodes in our program) on the JPanel, so that the next time the user can just load using this file, instead of having to wait for the program to parse the entire code base again.
